Hexo e o Nginx

Utilizando Hexo e Nginx

Como o Hexo utiliza o formato .EJS (Espécie de Javascript ejetável) para renderizar as páginas, é necessário utilizar uma junção com o Nginx para fazer com que o Servidor Linux use o Hexo como um provedor de páginas para o Web Server. Para isso, preciamos criar um deploy que irá fazer com que páginas geradas no Hexo sejam tranferidas para a pasta /www padrão do Nginx.

Criando Máquina virtual com o AWS

Porque AWS

Decidimos escolher o Aws por causa da sua praticidade, também a liçenca gratuita permite um bom nível de utilização. Mesmo que esse tipo de liçenca disponibilize apenas um tipo de configuração de Hardware dá maquina virtual, não houve nenhum problema por parte da Aws.

Configuração da Máquina Virtual

Escolhemos o Ubuntu Server 18.0.4 para hospedar o servidor. A configuração permitida era com 4x núcleos e apenas 1GB de Ram, porém serviu sem nenhum tipo de travamento para oque foi proposto. Criar a máquina foi simples e rápido.

Personalizações

Personalizações inicias realizadas no Blog

Escolhemos o tema Landscape por ser simples, elegante e também por ser recursivo. Para se adequar ao que queriamos, foi necessário alterar a linguagem dos cabeçalhos e sidebar para portugues, e também adicionamos comentários com o disqus, oque fez com que o site ficasse com um ‘rosto’ bem mais agradável ao nosso ver, pois quem visitar a página tem a possibilidade de fazer um comentário em cada blog entry e em cada página também.

Outras customizações realizadas foram coisas mais simples como os icones do github e a imagem da navbar do site.

Temas Hexo e suas possibilidades

O tema landscape não foi a primeira opção, pois haviamos escolhido um tema chamado Skapp, mas ele necessitava da instalação de um pacote chamado nodejiba, que apresentou problemas na instalação, então não podemos usar, pois esse pacote estava com problemas na instalação. Apensar de haver uma quantidade boa de temas, muitos não possuiam muita documentação ou tinham problemas com dependencias de terceiros.
